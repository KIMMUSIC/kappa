# [PHASE_1] 기초 하네스 방어선 및 단일 자가 치유 루프 구축

## 1. Phase 1의 목표 (Objective)
본 단계에서는 MCP 도구를 연동하거나 다중 에이전트를 구성하는 것을 **절대 금지**한다.
현재 단계의 유일한 목표는 에이전트가 통제력을 잃고 인프라를 파괴하거나 비용을 폭주시키는 현상을 원천 차단하기 위한 **'방어막(Guardrails)'**과 그 안에서 구동되는 **'결정론적 단일 자가 치유 루프'**를 최우선으로 구축하는 것이다.

## 2. 작업 지시 사항 (Action Items)

### Task 1: 예산 게이트 및 회로 차단기 (Budget Gate & Circuit Breaker)
LLM API 통신부 최상단에 재무적 방어선을 구축하라.
*   **요구사항:**
    *   세션별 최대 누적 토큰 수 또는 허용 비용(Cost) 한도를 추적하는 전역 상태 관리 모듈(또는 LangGraph State)을 구현할 것.
    *   LLM API 호출 전후로 누적 사용량을 갱신하며, 할당된 한도 초과 시 즉시 `BudgetExceededException`을 발생시켜 전체 프로세스를 강제 셧다운 시킬 것.

### Task 2: 결정론적 샌드박스 (Deterministic Sandbox) 환경 구축
에이전트가 생성한 코드나 쉘 스크립트가 안전하게 실행될 격리 환경 인터페이스를 작성하라.
*   **요구사항:**
    *   코드 문자열을 입력받아, 호스트 머신과 완벽히 격리된 일회성 Docker 컨테이너(또는 엄격히 제한된 파이썬 샌드박스) 환경에서 실행하는 클래스/함수를 작성할 것. (절대 호스트에서 직접 `subprocess`를 띄우지 말 것).
    *   실행이 완료되면 구조화된 형태로 `exit_code`, `stdout`, `stderr`를 반환해야 한다.
    *   무한 루프 코드가 샌드박스 자원을 고갈시키지 않도록 실행 타임아웃(Timeout) 제어를 필수적으로 포함할 것.

### Task 3: LangGraph 기반 단일 자가 치유(Self-Healing) 루프 조립
Task 1, 2의 인프라를 바탕으로 단일 에이전트가 코드를 짜고, 검증하고, 실패 시 스스로 수정하는 상태 머신을 구축하라.
*   **요구사항:**
    1.  **원자적 파서(Parser) 노드:** LLM의 출력이 `<think>` 와 `<action>`(실행할 코드) 블록으로 강제되도록 프롬프트를 구성하고, 포맷 위반 시 에러를 반환하는 필터 노드 구성.
    2.  **구문 빗자루(Linter) 노드:** 파싱된 코드를 샌드박스로 전송하기 전, 기본 문법 검사기(AST 등)를 통과시켜 치명적 구문 오류를 사전 차단.
    3.  **자가 치유 엣지(Conditional Edge):** 샌드박스 실행 결과 `exit_code != 0`일 경우, `stderr`(에러 트레이스)와 원본 목표를 결합하여 다시 Coder 노드로 루프백(Loop-back)하는 조건부 엣지 구현. 단, 무한 루프 방지를 위해 최대 재시도 횟수(Hard Limit) 설정 필수.

## 3. 완료 검증 기준 (Definition of Done)
코드를 작성한 후, 아래 3가지 시나리오를 통과하는 테스트 코드를 작성하여 스스로 동작을 증명해야 한다.
1.  **Budget Test:** 의도적으로 무한 루프를 유발했을 때, 설정된 예산 한도 도달 즉시 네트워크가 차단되고 시스템이 강제 종료되는가?
2.  **Sandbox Test:** 파괴적인 시스템 명령(`rm -rf` 등)을 주입했을 때, 호스트 PC가 완벽히 보호되며 에러 코드를 정상 반환하는가?
3.  **Self-Healing Test:** 의도적으로 문법/런타임 에러가 있는 코드를 생성하게 했을 때, 샌드박스 에러 로그를 피드백으로 받아 스스로 코드를 수정하여 최종적으로 성공(`exit_code == 0`)해 내는가?
