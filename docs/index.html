DAG 분해 전략

LLM에게 목표를 전달하면 depends_on으로 연결된 SubTask DAG를 JSON으로 반환. 최대 10개 서브태스크 제한.

Quality Gate

Reviewer 검수 루프

Reviewer LLM이 approve/reject 판정. 거부 시 critique가 워커에 주입. 누적 3회 초과 시 실패.

Telemetry

Agent-RRM 궤적 기록

think(추론), critique(비평), score(0.0~1.0)를 JSONL에 누적하여 사후 분석 지원.

Session Lane

Per-Key 자원 직렬화

SyncSessionLane이 동일 키 동시 접근 직렬화. 메타 Lock + 개별 Lock 2계층.

Jitter Backoff

Decorrelated Jitter

AWS 권장 알고리즘으로 Thundering Herd를 Jitter로 분산.

Super-Graph

계층적 그래프 패턴

OrchestratorGraph가 SelfHealingGraph를 블랙박스 워커로 소비. Additive-Only 설계.

Verification Pipeline
5-Layer Verification Pipeline

에이전트의 모든 출력은 5개의 결정론적 관문을 순차적으로 통과해야 합니다.

L1Atomic XML MatchingLLM 출력이 <think>/<action>/<tool_call> 구조를 준수하는지 검증parse_llm_output()
L2Syntactic BroomAST 파싱으로 문법 오류를 샌드박스 전송 전 사전 차단lint_code()
L3Semantic DefenseJaccard 유사도 기반 의미론적 반복 감지 및 조기 차단SemanticLoopDetector
L4Self-Healing실행 오류 시 stderr + 원본 목표로 자동 교정 루프conditional_edges
L5Runtime Smoke Test격리 컨테이너에서 제한 시간 내 동작 물리적 증명SandboxExecutor
Development Track
Implementation Timeline

통제 없는 자율성은 재앙이기 때문입니다. Phase 1에서 방어막을, Phase 2에서 인지적 자율성을, Phase 3에서 오케스트레이션을, Phase 4에서 프로덕션 연동을, Phase 5에서 실전 엔트리포인트를 구축했습니다.

Phase 1 — Deterministic Guardrails
T1Budget Gate & Circuit Breaker9 ✓▶

LLM API 통신부 최상단에 재무적 방어선 구축. 세션별 토큰/비용 이중 추적, 한도 초과 시 회로 차단기가 영구적으로 트립되어 모든 후속 API 호출을 물리적으로 차단합니다.

T2Deterministic Sandbox21 ✓▶

에이전트 생성 코드를 호스트 OS에서 절대 실행하지 않는 격리 환경. 일회성 Docker 컨테이너에서 메모리 제한, 네트워크 차단, 타임아웃을 강제하고 실행 후 즉시 파괴합니다.

T3Self-Healing Loop17 ✓▶

LangGraph 상태 머신. 코드 생성 → XML 파싱 → AST 린트 → 샌드박스 실행의 파이프라인에서 실패 시 에러 트레이스를 피드백으로 자동 재시도. 최대 재시도 횟수로 무한 루프를 방지합니다.

Phase 2 — Cognitive Autonomy
T4VFS Memory — 격리 파일시스템34 ✓▶

에이전트 전용 격리 작업 디렉토리에서 학습 내용을 기록하고 검색하는 VFSManager. 경로 이탈 공격(../)을 원천 차단하며, 핵심 파일은 시스템 프롬프트에 자동 주입됩니다.

T5MCP Tool Registry & Bridge30 ✓▶

BaseTool 프로토콜 기반 플러그 앤 플레이 도구 레지스트리. read_memory, write_memory 시스템 도구를 기본 제공하며, 모든 도구 실행 비용은 BudgetTracker를 경유하여 추적됩니다.

T6Semantic Loop Detector24 ✓▶

최근 N개의 think 내용과 도구 호출 인자의 Jaccard 유사도를 슬라이딩 윈도우로 계산. 동일한 헛발질이 임계치 이상 반복되면 조기 차단합니다.

T7Self-Healing V2: Tool Branching20 ✓▶

파서가 <action>과 <tool_call>을 분기하여 코드 실행(Sandbox)과 도구 호출(Tool)을 조건부 엣지로 분리. VFS 메모리 컨텍스트 자동 주입, 의미론적 감지기 통합으로 Graph V2 완성.

Phase 3 — Multi-Agent Orchestration
T8SessionLane & Jitter Backoff32 ✓▶

다중 워커의 자원 경합을 막는 Per-key 세마포어와 Thundering Herd 현상을 방지하는 Decorrelated Jitter 지수적 백오프. ThreadPoolExecutor와 asyncio 환경 모두 지원합니다.

T9Orchestrator Super-Graph39 ✓▶

거대 목표를 DAG로 분해하는 Planner, SelfHealingGraph 워커를 병렬 실행하는 Dispatcher, 산출물을 검수하는 Reviewer(Quality Gate), 결과를 통합하는 Finalizer로 구성된 LangGraph Super-Graph.

T10Agent-RRM Telemetry Pipeline32 ✓▶

워커의 추론 궤적(think), Reviewer 비평(critique), 종합 점수(score)를 JSONL로 누적 기록하는 옵저버 패턴 수집기. 스레드 안전 보장, 통계 요약(summary()) 제공.

Phase 4 — Production Integration
T11MCP Client Bridge & Adapter37 ✓▶

외부 MCP 서버의 도구를 MCPTransport Protocol로 발견하고, MCPToolAdapter가 BaseTool 규격으로 래핑하여 ToolRegistry에 일괄 등록. 모든 호출은 BudgetTracker 예산망을 반드시 경유합니다.

T12Domain RAG Pipeline50 ✓▶

문서를 청킹하고 임베딩하여 InMemoryVectorStore에 인덱싱하는 RAGManager. KnowledgeSearchTool이 BaseTool로 ToolRegistry에 등록되어 에이전트가 도메인 지식을 검색할 수 있습니다. EmbeddingProvider/VectorStore 모두 Protocol 기반 교체 가능.

T13Human-in-the-Loop & Rich CLI57 ✓▶

HITLPolicy가 파괴적 명령, 예산 경고, 과다 재시도를 탐지하면 HITLInterceptor가 Rich 프롬프트로 사용자에게 [Y/N] 승인을 요청. Rich Live 대시보드가 Plan Table, Activity Log, Budget Gauge를 실시간 렌더링합니다.

Phase 5 — Production Launch
T14Production Entrypoint & DI Root★▶

모든 프로덕션 컴포넌트를 하나의 DI Root(main.py)에서 조립하는 최종 엔트리포인트. AnthropicProvider, DockerRuntime, ToolRegistry, SemanticLoopDetector, TelemetryManager, HITLInterceptor를 실체화하고 OrchestratorGraph에 주입합니다. argparse로 --goal, --auto-approve, --max-tokens 등을 지원하며, 단일 실행 모드와 인터랙티브 REPL 모드를 모두 제공합니다.

Module Architecture
Fourteen Pillars of Kappa

각 모듈은 독립 프로토콜 기반으로 설계되어 테스트 시 Mock으로 완전 교체 가능합니다.

$

kappa.budget

BudgetTracker — 스레드 안전 토큰/비용 추적. RLock 기반.
BudgetGate — LLM 호출 래퍼. pre_check → call → record_usage.
LLMProvider — Protocol. AnthropicProvider 구현체 제공.

S

kappa.sandbox

SandboxExecutor — 코드 → Docker 컨테이너 실행.
ContainerRuntime — Protocol. DockerRuntime.
SandboxResult — exit_code, stdout, stderr.

G

kappa.graph

SelfHealingGraph V2 — 5-node LangGraph StateGraph. <action>/<tool_call> 분기 파서.
AgentState — TypedDict + tool_calls, memory_context.

M

kappa.memory

VFSManager — 격리 작업공간 파일 시스템.
Path Traversal — ../ 공격 원천 차단.

T

kappa.tools

ToolRegistry — 도구 등록/디스패치 + 예산 연동.
BaseTool — Protocol. 플러그 앤 플레이.

D

kappa.defense

SemanticLoopDetector — Jaccard 유사도 감지기.
SemanticLoopException — 조기 차단 예외.

I

kappa.infra

SyncSessionLane — Per-key Lock 자원 직렬화.
jitter_backoff — Decorrelated Jitter 백오프.

O

kappa.graph.orchestrator

OrchestratorGraph — LangGraph Super-Graph 오케스트레이터. Planner/Reviewer — LLM 기반 DAG 분해 & Quality Gate.
SubTask — TypedDict DAG 노드, 병렬 디스패치.

R

kappa.telemetry

TelemetryManager — JSONL 궤적 기록기.
TrajectoryRecord — think/critique/score.
summary() — 집계 통계.

M

kappa.tools.mcp

MCPBridge — 외부 MCP 서버 연결/탐색/호출. JSON-RPC 2.0 프로토콜.
MCPToolAdapter — MCP 도구를 BaseTool로 래핑. mcp:서버:도구 네임스페이스.
MCPTransport — Protocol. stdio/SSE 교체 가능.

K

kappa.rag

RAGManager — 청킹/임베딩/인덱싱 파이프라인.
KnowledgeSearchTool — BaseTool 준수 검색 도구.
InMemoryVectorStore — 코사인 유사도.

H

kappa.hitl

HITLPolicy — 예산/파괴적 명령/재시도 3종 트리거.
HITLInterceptor — 콜백 기반 승인 개입.
ApprovalPrompt — Protocol. 감사 추적.

C

kappa.cli

Rich Live Dashboard — Plan Table, Activity Log.
Budget Gauge — 실시간 토큰/비용 표시.
HITL Prompt — [Y/N] 승인 대화.

# Phase 3: 다중 에이전트 오케스트레이션

OrchestratorGraph ← SelfHealingGraph (Sub-Graph worker)
TelemetryManager ← OrchestratorGraph._reviewer_node

# Phase 4: 프로덕션 MCP/RAG/HITL 연동
MCPBridge        ← MCPToolAdapter → ToolRegistry
RAGManager       ← KnowledgeSearchTool → ToolRegistry
HITLInterceptor  ← OrchestratorGraph.approval_callback
Rich Live        ← OrchestratorGraph.stream()

# Phase 5: 프로덕션 런칭 — DI Root 조립
main.py           ← build_orchestrator() DI Root
                     모든 실체 컴포넌트를 조립 → run_dashboard()
