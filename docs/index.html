GEODE Reference Model
5-Layer Verification Pipeline

에이전트의 모든 출력은 5개의 결정론적 관문을 순차적으로 통과해야 합니다. 각 계층은 독립적 검증 책임을 가집니다.

L1 Atomic XML Matching LLM 출력이 <think>/<action> 구조를 준수하는지 검증 parse_llm_output()
L2 Syntactic Broom AST 파싱으로 문법 오류를 샌드박스 전송 전 사전 차단 lint_code()
L3 Symbolic Ripple 모듈 변경의 하위 의존성 파급 효과 자동 추적 Phase 2+
L4 Self-Healing 실행 오류 시 stderr + 원본 목표로 자동 교정 루프 conditional_edges
L5 Runtime Smoke Test 격리 컨테이너에서 제한 시간 내 동작 물리적 증명 SandboxExecutor

Task 2

Task 3

Self-Healing Loop

Task 1, 2의 인프라 위에 조립된 LangGraph 상태 머신. 코드 생성 → XML 파싱 → AST 린트 → 샌드박스 실행의 파이프라인에서 실패 시 에러 트레이스를 피드백으로 자동 재시도합니다. 최대 재시도 횟수로 무한 루프를 방지합니다.

Module Architecture
Three Pillars of Phase 1

각 모듈은 독립 프로토콜 기반으로 설계되어 테스트 시 Mock으로 완전 교체 가능합니다.

$

kappa.budget

BudgetTracker — 스레드 안전 토큰/비용 추적. RLock 기반.
BudgetGate — LLM 호출 래퍼. pre_check → call → record_usage.
LLMProvider — Protocol. AnthropicProvider 구현체 제공.

S

kappa.sandbox

SandboxExecutor — 코드 문자열 → Docker 컨테이너 실행.
ContainerRuntime — Protocol. DockerRuntime 구현체.
SandboxResult — exit_code, stdout, stderr, timed_out.

G

kappa.graph

SelfHealingGraph — LangGraph StateGraph 조립.
AgentState — TypedDict. 그래프 상태 정의.
nodes.py — parse_llm_output, lint_code, build_messages.
