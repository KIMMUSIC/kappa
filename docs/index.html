Phase 6 Deep Dive
Meta-Prompt Pipeline

사용자의 모호한 입력을 3단계 파이프라인으로 정밀 해석합니다. Meta-Prompter가 갭을 분석하고, Interview가 깊은 요구사항을 추출하며, Plan Approval이 실행 전 최종 관문이 됩니다.

01
META-PROMPTER
사용자의 모호한 입력을 분석하여 갭(목표, 범위, 기술 스택)을 식별하고 모호성을 0.0~1.0으로 정량화합니다. CoT/ReAct/Direct 중 최적 전략을 자동 선택합니다.
gap analysis ambiguity scoring
02
INTERVIEW
PM처럼 핵심 질문 3~5개를 도출하고, 답변을 종합하여 정밀한 "Golden Goal"을 합성합니다.
conditional
ambiguity > threshold → Interview 활성화
low ambiguity → skip to Planner
03
GOLDEN GOAL
인터뷰 답변과 원본 입력을 종합하여 구조화된 정밀 요구사항을 합성합니다. 모든 갭이 채워진 실행 가능한 목표입니다.
synthesized requirement
04
PLANNER
Golden Goal을 DAG(방향성 비순환 그래프)로 분해합니다. 각 SubTask에 depends_on 의존성을 설정하여 병렬 실행 가능한 계획을 수립합니다.
DAG decomposition
05
PLAN APPROVAL
Planner가 생성한 DAG 계획을 사용자에게 제시합니다. Y(승인)로 Dispatcher 실행, N(거부)으로 중단, M(수정)으로 재계획합니다.
Y: approve N: reject
↺ M: modify → 피드백과 함께 Planner로 재계획

Stage 1 — Meta-Prompter

Gap Analysis & Strategy Selection

사용자 입력의 모호성을 0.0~1.0으로 정량화하고, 누락된 정보(목표, 범위, 기술 스택 등)를 식별합니다. CoT/ReAct/Direct 중 최적 전략을 자동 선택하여 프롬프트를 구조화합니다.

Stage 2 — Interview

Rich CLI PM Panel

모호성이 높을 때 활성화되는 조건부 인터뷰 엔진. PM처럼 핵심 질문 3~5개를 도출하고, 답변을 종합하여 정밀한 "Golden Goal"을 합성합니다. 낮은 모호성이면 건너뜁니다.

Stage 3 — Plan Approval

Human-Gated Execution

Planner가 생성한 DAG 계획을 사용자에게 제시합니다. Y(승인)로 즉시 실행, N(거부)으로 중단, M(수정)으로 Planner에 피드백을 전달하여 재계획합니다.

Verification Pipeline
5-Layer Verification Pipeline

에이전트의 모든 출력은 5개의 결정론적 관문을 순차적으로 통과해야 합니다.

L1Atomic XML MatchingLLM 출력이 <think>/<action>/<tool_call> 구조를 준수하는지 검증parse_llm_output()
L2Syntactic BroomAST 파싱으로 문법 오류를 실행 전 사전 차단lint_code()
L3Semantic DefenseJaccard 유사도 기반 의미론적 반복 감지 및 조기 차단SemanticLoopDetector
L4Self-Healing실행 오류 시 stderr + 원본 목표로 자동 교정 루프conditional_edges
L5Runtime Smoke TestAST 안전 검증 후 호스트에서 제한 시간 내 동작 증명HostExecutor
Development Track
Implementation Timeline

통제 없는 자율성은 재앙이기 때문입니다. Phase 1에서 방어막을, Phase 2에서 인지적 자율성을, Phase 3에서 오케스트레이션을, Phase 4에서 프로덕션 연동을, Phase 5에서 실전 엔트리포인트를 구축했습니다.

Phase 1 — Deterministic Guardrails
T1Budget Gate & Circuit Breaker9 ✓▶

LLM API 통신부 최상단에 재무적 방어선 구축. 세션별 토큰/비용 이중 추적, 한도 초과 시 회로 차단기가 영구적으로 트립되어 모든 후속 API 호출을 물리적으로 차단합니다.

T2Host Executor + Safety27 ✓▶

Docker 컨테이너 대신 호스트에서 직접 실행. AST 기반 SafetyValidator가 위험 패턴(os.system, subprocess, exec/eval, shutil.rmtree)을 사전 차단하고, HITL 승인 콜백이 실행을 게이트합니다. subprocess.run으로 타임아웃을 강제하며, output_dir을 CWD로 설정하여 생성 파일을 영속화합니다.

T3Self-Healing Loop17 ✓▶

LangGraph 상태 머신. 코드 생성 → XML 파싱 → AST 린트 → 샌드박스 실행의 파이프라인에서 실패 시 에러 트레이스를 피드백으로 자동 재시도. 최대 재시도 횟수로 무한 루프를 방지합니다.

Phase 2 — Cognitive Autonomy
T4VFS Memory — 격리 파일시스템34 ✓▶

에이전트 전용 격리 작업 디렉토리에서 학습 내용을 기록하고 검색하는 VFSManager. 경로 이탈 공격(../)을 원천 차단하며, 핵심 파일은 시스템 프롬프트에 자동 주입됩니다.

T5MCP Tool Registry & Bridge30 ✓▶

BaseTool 프로토콜 기반 플러그 앤 플레이 도구 레지스트리. read_memory, write_memory 시스템 도구를 기본 제공하며, 모든 도구 실행 비용은 BudgetTracker를 경유하여 추적됩니다.

T6Semantic Loop Detector24 ✓▶

최근 N개의 think 내용과 도구 호출 인자의 Jaccard 유사도를 슬라이딩 윈도우로 계산. 동일한 헛발질이 임계치 이상 반복되면 조기 차단합니다.

T7Self-Healing V2: Tool Branching20 ✓▶

파서가 <action>과 <tool_call>을 분기하여 코드 실행(Sandbox)과 도구 호출(Tool)을 조건부 엣지로 분리. VFS 메모리 컨텍스트 자동 주입, 의미론적 감지기 통합으로 Graph V2 완성.

Phase 3 — Multi-Agent Orchestration
T8SessionLane & Jitter Backoff32 ✓▶

다중 워커의 자원 경합을 막는 Per-key 세마포어와 Thundering Herd 현상을 방지하는 Decorrelated Jitter 지수적 백오프. ThreadPoolExecutor와 asyncio 환경 모두 지원합니다.

T9Orchestrator Super-Graph39 ✓▶

거대 목표를 DAG로 분해하는 Planner, SelfHealingGraph 워커를 병렬 실행하는 Dispatcher, 산출물을 검수하는 Reviewer(Quality Gate), 결과를 통합하는 Finalizer로 구성된 LangGraph Super-Graph. 의존 Task 결과를 후속 워커에 자동 주입하고, Reviewer가 Workspace 파일 내용을 검증합니다.

T10Agent-RRM Telemetry Pipeline32 ✓▶

워커의 추론 궤적(think), Reviewer 비평(critique), 종합 점수(score)를 JSONL로 누적 기록하는 옵저버 패턴 수집기. 스레드 안전 보장, 통계 요약(summary()) 제공.

Phase 4 — Production Integration
T11MCP Client Bridge & Adapter37 ✓▶

외부 MCP 서버의 도구를 MCPTransport Protocol로 발견하고, MCPToolAdapter가 BaseTool 규격으로 래핑하여 ToolRegistry에 일괄 등록. 모든 호출은 BudgetTracker 예산망을 반드시 경유합니다.

T12Domain RAG Pipeline50 ✓▶

문서를 청킹하고 임베딩하여 InMemoryVectorStore에 인덱싱하는 RAGManager. KnowledgeSearchTool이 BaseTool로 ToolRegistry에 등록되어 에이전트가 도메인 지식을 검색할 수 있습니다. EmbeddingProvider/VectorStore 모두 Protocol 기반 교체 가능.

T13Human-in-the-Loop & Rich CLI57 ✓▶

HITLPolicy가 파괴적 명령, 예산 경고, 과다 재시도를 탐지하면 HITLInterceptor가 Rich 프롬프트로 사용자에게 [Y/N] 승인을 요청. Rich Live 대시보드가 Plan Table, Activity Log, Budget Gauge를 실시간 렌더링합니다.

Phase 5 — Production Launch
T14Production Entrypoint & DI Root★▶

모든 프로덕션 컴포넌트를 하나의 DI Root(main.py)에서 조립하는 최종 엔트리포인트. AnthropicProvider, HostExecutor, ToolRegistry, SemanticLoopDetector, TelemetryManager, HITLInterceptor를 실체화하고 OrchestratorGraph에 주입합니다. argparse로 --goal, --auto-approve, --max-tokens 등을 지원하며, 단일 실행 모드와 인터랙티브 REPL 모드를 모두 제공합니다.

T15CWD Workspace Mount & Git Safety1 ✓▶

호스트 CWD를 workspace_dir로 지정하여 프로젝트 파일을 직접 참조합니다. _scan_workspace()가 프로젝트 파일을 재귀 탐색(깊이 3, 최대 80항목)하여 Planner에게 파일 컨텍스트를 주입하고, Worker 프롬프트에 호스트 경로를 안내합니다. Git stash 기반 안전망이 실행 전 스냅샷을 생성하고, 성공 시 diff를 표시하며, 실패 시 자동 복원합니다.

Phase 6 — Meta-Prompt Pipeline
T16Meta-Prompter — Gap Analysis & Prompt Structurization13 ✓▶

사용자 입력의 모호성을 0.0~1.0으로 정량화하는 MetaPrompter. 목표/범위/기술 스택/제약 조건 등의 갭을 탐지하고, 모호성 점수에 따라 CoT(Chain of Thought), ReAct, Direct 중 최적 전략을 자동 선택합니다. parse_meta_prompt_response()가 LLM 응답을 정규화합니다.

T17Interview Engine — Rich CLI PM Panel5 ✓▶

모호성이 높을 때 활성화되는 조건부 PM 인터뷰 엔진. MetaPrompter가 탐지한 갭을 기반으로 핵심 질문 3~5개를 도출하고, INTERVIEW_SYNTHESIZER_PROMPT를 통해 답변을 종합하여 정밀한 "Golden Goal"을 합성합니다.

T18Orchestrator Meta — Sentinel Protocol & Graph Integration17 ✓▶

MetaPrompter와 InterviewEngine을 OrchestratorGraph의 진입점으로 통합하는 Sentinel Protocol. _meta_prompt_node가 모호성을 평가하고, 필요 시 Interview를 활성화하며, Plan Approval 노드가 Y/N/M 게이트로 실행을 제어합니다. 17개 통합 테스트로 전체 플로우를 검증합니다.
